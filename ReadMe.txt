**アピールポイント**

・操作が簡単に、そして強すぎず弱すぎない敵を目指しました。
  ゲームの目的は敵よりも多くの木の実を集めるというシンプルなものに
  なっています。
・制作時にバグや仕様変更をしやすくするためにプレイヤーと敵に
  StatePatternを使用しました。
  使用理由は動いた時の状態毎にクラスに分け管理することで、
  バグの修正や動きの調整をする時に、どこを変更したらよいのかを
  一目でわかりやすくするために使用しました。
・コード量を減らすためにGameObjectを使用しました。
  ゲーム上で出てくるオブジェクトは全てGameObjectクラスを継承し、
  GameObjectManagerクラスで管理しています。
  これにより、UpdateやRenderといった更新処理や描画処理を一括で管理出来るので、
  オブジェクト毎に書く必要がなくなり、コード量を減らすことが出来ました。
